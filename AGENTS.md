# AGENTS

- 言語: 語尾は「にゃ」で
- 計画: 設定変更等の軽微なもの以外、計画及び計画承認のない実装はしないこと。追加仕様発生時は、計画全体を見直して修正すること。
- 変更範囲: 要求された箇所のみ。不要なリネーム/整形/設計変更は禁止
- 非ミッションクリティカル前提: 過度に凝った作り込みはしない
- コーディング: 同じ意味の定数や設定、関数、モデルを多重定義しない。保守性最重要視！MVCモデルは必要か自問せよ。あらゆる構造は極力シンプルに。定数直埋めは極力避ける。
- GUI: 誰が使用しても理解できるUIを心がけよ。ボタン等の名称は操作対象の明確化！
- コメント: 関数内コメントは日本語で丁寧に。関数のドキュメントコメントは必須
- I/O方針: 過剰な保存・再読込を避ける。差分保存を優先。
- I/O、通信コスト: 可能な限り下げる
- 品質: 軽微な設定変更や資料変更以外、ビルド/テストを通してから完了報告する
- Git: コミット/プッシュはユーザーが行う。Codexは実行しない
- 互換性: 基本的に考慮しなくてよい
- 暗号化: APIキーなど漏洩すると問題あるものは暗号化する
